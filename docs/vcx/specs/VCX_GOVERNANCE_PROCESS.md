# VCX Governance Process (v1)

**Status**: active
**Owner**: Core Runtime
**Last reviewed**: 2026-02-20

## Purpose

Define the governance and decision process for VCX standard evolution.

## Working Group Model

- Chair: Core Runtime representative
- Required reviewers: Security, Runtime, at least one external implementer (when available)
- Meeting cadence: bi-weekly

## Decision Classes

- Editorial: wording/clarity only
- Additive: backward-compatible additions
- Breaking: wire or semantic change

## Decision Rules

- Editorial/additive: simple majority of attending reviewers.
- Breaking: unanimous approval by Core Runtime + Security + Governance chair.

## Proposal Lifecycle

1. Open proposal with problem statement and compatibility impact.
2. Include conformance update plan.
3. Review and decision record.
4. Merge with version bump and changelog update.

## Transparency Requirements

- Every accepted change must have a public decision note.
- Every rejected breaking change must include rationale.
